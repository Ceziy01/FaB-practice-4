# 🎮 Магазин видеоигр

# Практика 1 
Препроцессоры использовались для стилизации карточек товаров, цветов и шрифтов. Конкретно в данном приложении использовался SCSS.

# Практика 2
Серверная часть на писана на языке JavaScript с помощью Node.js и Express. Она отвечает за обработку http запросов и возвращает данные в формате json.

# Практика 3
Для обмена данными между сервером и клиентской частью используется формат json

# Практика 4
Серверная часть на писана на языке JavaScript с фреймворка React. С его помощью реализована страница магазин, интерфейс и карточки товаров.

# Практика 5
Для создания документации был использован Swagger. Он позволяет автоматически генерировать интерактивную документацию и тестировать эндпоинты прямо в браузере.

# Описание проекта

Приложение представляет собой простой CRUD-магазин видеоигр.

Пользователь может:

- Добавлять товар
- Просматривать список товаров
- Получать товар по ID
- Обновлять товар
- Удалять товар

Все операции выполняются через REST API.

# Frontend (React)

Клиентская часть реализована на **React**.

## Используется:

- `useState` — управление состоянием
- `useEffect` — загрузка данных при монтировании
- `axios` — HTTP-запросы
- SCSS — стилизация

## Функционал:

- Отображение списка товаров
- Форма добавления товара
- Удаление товара без перезагрузки страницы
- Динамическое обновление интерфейса

---

# Backend (Express)

Серверная часть реализована на **Express.js**.

## Используется:

- express — создание REST API
- cors — разрешение кросс-доменных запросов
- nanoid — генерация уникальных ID
- swagger-jsdoc — генерация OpenAPI документации
- swagger-ui-express — интерфейс документации

## Реализованные эндпоинты:

| Метод | Endpoint | Описание |
|--------|----------|----------|
| GET | /api/products | Получить список товаров |
| GET | /api/products/:id | Получить товар по ID |
| POST | /api/products | Создать товар |
| PATCH | /api/products/:id | Обновить товар |
| DELETE | /api/products/:id | Удалить товар |
